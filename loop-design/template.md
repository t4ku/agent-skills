## 実行プリミティブ (Claude Code 2.1.185+)

> ⚠️ `claude --goal` / `--max-iterations` は**現行 Claude Code に存在しない** (2.1.185 で確認、2026-06)。
> 本 spec は **設計図**として使い、下記いずれかのプリミティブに落とす。タスク特性で選ぶ。

| タスク特性 | プリミティブ | 7 要素の充足 |
|----------|-----------|------------|
| 多段 / 決定的 / 並列検証 / adversarial | **A. `Workflow` ツール** | budget/pipeline/parallel がネイティブ |
| 単一エージェントで「完了まで反復」 | **B. `/loop`** or **C. `ralph-loop`** | 反復再投入 + completion gate |
| 定期実行 (cron 的) | **B. `/loop {{interval}} /cmd`** or `schedule` skill | 間隔起動 |
| 1 ショット非対話 | **D. `claude -p`** | 単発 |

### A. Workflow ツール (推奨: 多段 / adversarial)

7 要素の多くがツール側にある:
- **Cost guard (F6)** → `budget.total` / `budget.remaining()` (ハード上限、超過で throw)
- **Termination** → loop-until-budget / loop-until-count / dry-rounds パターン
- **Verification gates** → `pipeline()` のステージ + 決定的チェックは素の JS
- **Adversarial verifier (F5)** → `parallel()` で N 体 skeptic + `agentType` で別プロバイダ
- **State** → 構造化リターン + {{state_files}}

→ 本 spec の End Goal / Gates / Termination を `Workflow` の script に転記して起動。
（検証を item ごとに流すなら `pipeline()`、全件揃えてから処理するときだけ `parallel()` barrier）

### B. /loop skill (単一反復 / 定期)

```text
/loop /<your-slash-command>          # interval 省略 = self-paced (ScheduleWakeup で間隔自動)
/loop {{interval}} /<your-slash-command>   # 定期実行
```
- 終了条件は prompt 内に明示。completion を自己申告させず **output で証明** (F9)

### C. ralph-loop plugin (A 型: rigid な completion gate)

```text
/ralph-loop "<task>" --max-iterations {{max_turns}} --completion-promise "<完了時のみ出力する文>"
```
- completion-promise は **完全に TRUE のときのみ** 出力 (F9 偽合言葉対策をプラグインが強制)
- 旧 `claude --goal --max-iterations` の最も近いドロップイン代替

### D. 1 ショット非対話

```bash
claude -p "$(cat <<'SPEC'
... (本 spec)
SPEC
)"
```
